  FEATURES & UI — New Additions
  ─────────────────────────────
  • Implemented Progressive Mode.
    Intelligent scanning and locking of the ideal volume levels. Perfect for
    scenarios where real-time adjustment isn't desired.

  • Added EQ Graph System.
    Visual representation and manual control of the 5-band equalizer for
    surgical precision.

  • Added Media Controls.
    Native browser media keys support for play/pause, next, and previous track.

  • Bass Perceptual Compensation.
    Improved bass clarity at lower volumes using psychoacoustic modeling to
    compensate for the Fletcher-Munson curve.

  • New Visual Identity.
    Complete logo overhaul and general theme shift from red to a premium
    White/Silver "Moon" aesthetic.

  • Landing Page Prototype.
    Initial release of the professional, minimalist product landing page.

  • Firefox Support (Work in Progress).
    Underlying architecture adjustments to support Manifest v2/v3 compatibility
    for the Firefox version.

  BUG FIXES
  ─────────
  • Fixed Volume Jumping.
    Refined EMA (loudness average) logic to eliminate sudden jumps between
    tracks with different masters.

  AUDIO ENGINE — Bug Fixes
  ────────────────────────
  • Fixed extension randomly stopping during playback.
    AudioContext could get stuck in 'suspended' state with no recovery path.
    Now detects 5 consecutive suspended health-check ticks (~10s) while the
    video is playing, forces teardown and reattaches automatically.

  • Fixed volume jumping suddenly between tracks.
    Three compounding root causes identified and fixed:
      1. EMA (loudness average) was resetting after 600ms of silence — too short
         for normal gaps between tracks. Threshold raised to 3000ms.
      2. A fast-path block for large gain errors was speeding up gain changes
         when they should be slow, making transitions audible as a jump.
         Block removed entirely.
      3. Track-change boost used an 80ms smoothing constant — fast enough to
         hear as a jolt. Raised to 250ms for a natural fade.

  • Fixed automation queue stacking on audio params.
    Calling setTargetAtTime every 50ms without canceling previous events caused
    undefined browser behavior. Replaced all param automation with _smoothParam()
    which calls cancelAndHoldAtTime before scheduling new values.

  • Fixed graph reconnection happening on every apply() call.
    Was causing micro-glitches every time a slider was moved. Now cached via
    _currentRoute — only reconnects when bypass/processing mode actually changes.

  • Fixed reattach race condition.
    context.close() is async but reattach was immediate. Added 50ms buffer on
    all teardown → reattach paths to avoid attaching to a closing context.

  • Added video 'play' event listener to trigger a health check immediately
    when playback resumes, catching context issues before the 2s polling cycle.

  AUDIO ENGINE — Improvements
  ────────────────────────────
  • Default response time reduced: 1500ms → 800ms. Faster, less noticeable lag.
  • EMA blend on silence reset: blends 50/50 with the first new sample instead
    of dropping the average entirely, for a smoother re-entry after silence.
  • _teardown() now fully resets all transient state (_peakEnvelope,
    _silenceTicks, _trackChangeBoost, _currentCrestDb, _currentAdaptiveRatio…).

  ENGINE DE ÁUDIO — Correções
  ────────────────────────────
  • Corrigido: extensão parando do nada durante a reprodução.
    O AudioContext podia ficar preso no estado 'suspended' sem nenhuma
    recuperação. Agora detecta 5 ticks consecutivos suspensos (~10s) enquanto
    o vídeo está tocando, força o teardown e reconecta automaticamente.

  • Corrigido: volume pulando entre músicas de forma abrupta.
    Três causas raiz identificadas e corrigidas:
      1. O EMA (média de loudness) resetava após 600ms de silêncio — curto
         demais para pausas normais entre faixas. Limite elevado para 3000ms.
      2. Um bloco de atalho para erros de gain grandes acelerava as mudanças
         quando deveriam ser lentas, tornando as transições audíveis como pulo.
         Bloco removido por completo.
      3. O boost de troca de faixa usava constante de 80ms — rápido o
         suficiente para ser percebido. Elevado para 250ms para um fade natural.

  • Corrigido: acúmulo de automação na fila dos parâmetros de áudio.
    Chamar setTargetAtTime a cada 50ms sem cancelar eventos anteriores causava
    comportamento indefinido no browser. Toda automação substituída por
    _smoothParam(), que chama cancelAndHoldAtTime antes de agendar novos valores.

  • Corrigido: reconexão do grafo acontecendo em todo apply().
    Causava micro-glitches toda vez que um slider era movido. Agora cacheado via
    _currentRoute — só reconecta quando o modo bypass/processamento muda de fato.

  • Corrigido: race condition no reattach.
    context.close() é assíncrono mas o reattach era imediato. Adicionado buffer
    de 50ms em todos os caminhos de teardown → reattach.

  • Adicionado listener de evento 'play' no vídeo para disparar um health check
    imediatamente ao retomar a reprodução, antes do ciclo de polling de 2s.

  ENGINE DE ÁUDIO — Melhorias
  ────────────────────────────
  • Tempo de resposta padrão reduzido: 1500ms → 800ms. Mais rápido e menos
    perceptível.
  • Blend do EMA ao sair do silêncio: mistura 50/50 com a primeira amostra nova
    em vez de zerar a média, para uma re-entrada mais suave.
  • _teardown() agora reseta todo o estado transitório (_peakEnvelope,
    _silenceTicks, _trackChangeBoost, _currentCrestDb, _currentAdaptiveRatio…).
